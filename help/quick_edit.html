Quick edit lets you work on a node's body using an external editor. When you have this setting enabled for a given node, the contents of the node body will be read from the file, rather than the database. This means that you don't have to keep saving the node, viewing the node and so forth to see how your tweaks are going. This is meant to be an aid during the migration of pages only, and not for ongoing maintenance.

How it works

  • Anytime you save a node, which has Source to Node enabled on it, a quick edit file will also be saved with the contents of the enabled fields to public://source2node/[nid]/[field_name].

  • Anytime the node is loaded, the content of the Body Insert Field will be injected with the content of the quick edit file, rather than the database.

How to use it

  1. Load the file in an external browser; see the node edit form for the exact path.
  2. Point your browser to the node view page and make edits to the file using the external editor.
  3. Save the file in your editor and refresh the page, you will see the changes to the node without having to edit the node.
  4. Repeat as needed.
  5. When you're satisfied, you just need to enter the edit mode of the node and click save; whatever is in the file will be captured to the node when it's saved.
